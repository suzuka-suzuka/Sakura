为什么是 Sakura

把机器人运行时和群功能生态收拢到一个顺手的工作台。

框架负责 WebSocket 连接、事件分发、插件生命周期、Redis 状态和 Web 配置；综合插件提供 AI 对话、生图、Pixiv、表情、经济、钓鱼、群管和自动推送等常用能力。

Core

框架能力

Connect

多连接适配

支持 OneBot 正向/反向 WebSocket，也能接入 Milky HTTP/WebSocket，方便按部署环境切换。

Plugin

插件热加载

自动加载 `plugins/*/apps` 下的插件类，注册 Command、OnEvent 和 Cron 时保持业务代码清爽。

Panel

Web 配置面板

启动后提供可视化配置入口，可编辑框架、账号、插件菜单和动态选项。

State

Redis 状态存储

会话、上下文、定时任务和插件状态交给 Redis，适合长期运行的群机器人。

Economy

统一扣费钩子

插件 handler 可以声明消耗配置，由框架统一检查、扣费和按结果退款。

Ops

PM2 守护运行

内置生产启动、停止和日志脚本，部署后更容易观察和维护。

Sakura Plugin

综合插件让机器人像一个功能工作室。

Sakura Plugin 是可额外部署的综合功能插件，可安装到 `plugins/sakura-plugin`，围绕群聊互动、内容创作、自动推送和群管理提供完整配置 schema，推荐通过 Web 面板编辑。

Flow

运行链路

1

连接 QQ 客户端

通过 NapCat、Lagrange 或其他 OneBot/Milky 实现接入消息事件。

2

事件进入框架

Sakura 负责连接管理、事件封装、权限判断和插件分发。

3

插件响应群聊

插件按命令、事件或定时任务处理请求，并把结果发回群聊。

Start

三步跑起来

准备 Node.js 20、pnpm、Redis、Chrome/Chromium 和一个兼容 OneBot 或 Milky 的 QQ 客户端。

git clone https://github.com/suzuka-suzuka/Sakura.git
cd Sakura
pnpm install
pnpm dev
